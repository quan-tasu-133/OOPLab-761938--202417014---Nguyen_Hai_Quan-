10. Question:
If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?
Answer:
No, Java does not allow this. Because both constructors would have the same parameter type (String), Java cannot distinguish between them. Constructor overloading requires different parameter lists (either in number or type).
15.1.Question:
After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two objects still remain?
Answer:
The titles of the two objects remain unchanged because Java uses pass by value. When the method swap is called, the references to the objects are copied into the parameters o1 and o2. Swapping these copied references does not affect the original references in the main method. Therefore, the objects are not actually swapped.
15.2.Question:
After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is the title of the jungleDVD changed?
Answer:
The title of jungleDVD is changed because the method modifies the internal state of the object. Although Java passes parameters by value, the value of an object variable is a reference to the object. Therefore, the method can use this reference to change the object's attributes, such as its title.
Conclusion:
Java is always a pass by value programming language.
